# Домашнее задание к занятию "«Репликация и масштабирование. Часть 1»" - `Мохов Антон`
## Задание 1
На лекции рассматривались режимы репликации master-slave, master-master, опишите их различия.

Ответить в свободной форме.
## Ответ 
Главное различие заключается в направлении передачи данных и ролях серверов:
1. Направление данных:
Master-Slave (Мастер-Слейв): Односторонняя репликация. Данные идут от основного сервера (Master) к вспомогательному (Slave).
Master-Master (Мастер-Мастер): Двусторонняя репликация. Данные копируются в обе стороны, каждый сервер является одновременно и мастером, и слейвом для другого.
2. Запись и чтение:
Master-Slave: Все изменения данных (INSERT, UPDATE, DELETE) происходят только на Master. С сервера Slave данные обычно только читаются (SELECT) для распределения нагрузки.
Master-Master: Запись данных (изменения) может производиться на любой из серверов. Изменения автоматически синхронизируются со вторым сервером.
3. Применение:
Master-Slave: Чаще используется для масштабирования операций чтения и создания резервных копий (отказоустойчивость).
Master-Master: Используется для горячего резервирования (если один упал, второй продолжает принимать записи) и распределения нагрузки на запись (с осторожностью из-за конфликтов).
